# A2A v2 — Implementation Spec

Date: 2026-07-10
Status: Approved (v2 minimal, user-confirmed)

This spec merges the SERVER plan and CLIENT plan into one ordered, deduplicated
implementation checklist. It is scoped strictly to the three approved items and
the "rip out Monitor/arm to zero" cleanup.

---

## Goal

Cut A2A over from a live-push / auto-revive / Monitor-armed model to a **manual,
filesystem-oriented v2**:

1. **Discovery** becomes rich — `a2a list` and a new `a2a whois <lib_id>` tell an
   agent *where* another agent lives (identity summary, PARA dir, each session's
   title + transcript `.jsonl` path) so it can read that agent's dir/session
   directly (agents already have filesystem access).
2. **Handoff** becomes a pure enqueue — `a2a send` writes the envelope into the
   target's maildir and returns. No liveness gate, no warm nudge, no cold
   auto-revive. The human later opens the target agent and runs
   `a2a inbox --drain` manually.
3. **Monitor/arm is removed to zero** — the `arm` / `watch-cmd` CLI verbs, their
   lib helpers, the `POST /a2a/arm` route, both kicker constants, and the
   arm-on-boot section of the general agent prompt all go away. `AGENT_PROMPT_VERSION`
   bumps `v8 -> v9` so on-disk `general.md` migrates on next boot.

---

## Scope (the 3 approved items)

1. **DISCOVERY (full).** Enrich `a2a list` and add `a2a whois <lib_id>`. Both must
   surface, per agent: `description` (first paragraph of the agent's `identity.md`,
   if present), the agent's PARA `dir` path (`~/Projects/<x>`, `~/Areas/<x>`,
   `~/Resources/<x>`; `global` -> `~/`), each session's `title` (from the meta
   sidecar) alongside its `session_id`, and each session's transcript `.jsonl`
   path. `whois` additionally returns the FULL `identity.md` and ALL sessions
   (not just live ones).

2. **HANDOFF (manual drain).** `a2a send` becomes a pure enqueue: validate + build
   envelope + dedup + enqueue into the target maildir, then return
   `delivery="enqueued"`. Remove the cold-target auto-revive, kicker, and
   warm-nudge entirely. `inbox` / `read` / `drain` are unchanged.

3. **RIP OUT Monitor/arm to zero.** Remove `arm` + `watch-cmd` CLI subcommands and
   their lib functions; remove `POST /api/orchestrator/a2a/arm` and both kicker
   constants; cut the arm-on-boot A2A section from the general prompt and bump the
   version; rewrite `SKILL.md`; de-arm `bootstrap.sh`. Keep the `a2a_enabled` flag
   gate intact.

### Resolved design decisions (were open questions in the source plans)

- **whois transport:** `GET /api/orchestrator/a2a/whois?lib_id=<lib_id>` (query
  param). Avoids URL-encoding the slash in `areas/Home`; CLI `http_get_whois`
  matches. Open GET, no token (it is a directory, not a secret).
- **delivery string:** single value `"enqueued"` for all sends. `--session`
  targeting still routes the envelope to the session sub-maildir but does **not**
  change the delivery string. SKILL §4 output contract uses `"enqueued"`.
- **`--session` survives v2:** kept. It still works as an enqueue into the session
  sub-maildir; only the *liveness validation* (the old `not live` 400) is removed.
- **description extraction:** strip a leading markdown `# Heading` line, then take
  the first non-empty prose paragraph (text up to the first blank line).
- **cold agent `dir`:** derived from the `lib_id` kind even with zero sessions
  (`areas/Home` -> `~/Areas/Dom`). `global` / `None` -> `~/`.
- **transcript path source:** resolved server-side via `jsonl_mod.jsonl_path(sid)`.
  No change to `orchestrator_jsonl._build_summary` / `list_sessions` is needed
  (this closes a risk the client plan raised about `_build_summary` lacking a
  `path` field).
- **session dict key name:** `transcript` (the absolute `.jsonl` path) in both
  `list_agents` live sessions and `whois` sessions.
- **`list` verbosity:** the human-readable table is enriched (description + dir +
  per-session title/session_id/transcript path), per the approved scope that
  "both must surface" these fields. The `>1 live session` gate is dropped so
  titles/paths always print.

## Ordered change list (per file)

Build order is dependency-first: server data layer -> server routes -> prompt

### 1. `src/orbit/orchestrator_a2a.py` — data layer (do first)

Everything else consumes what this module emits. Keep the module import-safe
(no import of `orchestrator.py`; inline paths under `HOME` rather than importing
`agent_prompts` to preserve the self-contained contract).

1.1 **ADD helper `_para_dir_for(lib_id) -> str`** (near `_agent_name_for`, ~line
535). Map kind -> Capitalized PARA root: `areas -> Areas`, `projects -> Projects`,
`resources -> Resources`; return `str(HOME/<Root>/<rest>)`. `global` / `None` ->
`str(HOME)` (i.e. `~/`). `HOME` is already imported from `.discovery`.
**Capitalization is mandatory** — lib_id kinds are lowercase but on-disk roots
are Capitalized; a naive `HOME/kind/rest` join yields the wrong path.

1.2 **ADD helper `_identity_for(lib_id, *, full=False) -> str`** (near
`_agent_name_for`). Resolve `HOME/'.orchestrator'/'agents'/kind/rest/'identity.md'`
(inline to stay import-cycle-free; mirrors `agent_prompts.agent_identity_path`).
Read the file; when `full=False` return the FIRST paragraph (text up to the first
blank line, stripped of a leading `# ` heading); when `full=True` return the whole
file. Return `''` on missing file / bad lib_id / global. Used by both `list_agents`
(`description`) and `whois` (full `identity`).

1.3 **ENRICH `list_agents()`** (def at ~548; accumulation loop ~598-635; finalize
~639-642):
- While iterating `summaries`, build a `summ_by_sid` map (currently only
  `overlay`/`all_meta` is retained) so per-session titles can be resolved.
- For each agent group, add `description = _identity_for(lib_id)` (first para) and
  `dir = _para_dir_for(lib_id)`.
- In the finalize loop that emits `sessions` as `[{session_id, last_active}]`
  (LIVE-only), enrich each entry with `title = meta_mod.resolve_title(overlay.get(sid), summ_by_sid.get(sid))` and `transcript = str(jsonl_mod.jsonl_path(sid))`.
- **Keep** existing keys (`lib_id`, `name`, `warm`, `session_id`, `last_active`,
  and the LIVE-only `sessions` list) for back-compat. The `sessions` list stays
  LIVE-only here; the full (incl. cold) session set lives only in `whois`.

1.4 **ADD `whois(lib_id, live_session_ids: set[str]) -> dict`** (after
`list_agents`, ~line 646, end of file). Reuse the `list_sessions` + `all_meta`
grouping but for a SINGLE lib_id (normalize via `agent_key` / `GLOBAL_KEY`).
Return:
```
{
  "lib_id":   <normalized key>,
  "name":     _agent_name_for(...),
  "dir":      _para_dir_for(...),
  "identity": _identity_for(..., full=True),
  "warm":     <any session live>,
  "sessions": [ {session_id, title, last_active,
                 live: sid in live_session_ids,
                 transcript: str(jsonl_mod.jsonl_path(sid))}, ... ]
                 # ALL sessions (not live-only), sorted by last_active desc
}
```
For an unknown-but-valid lib_id, still return a not-found shape that includes
`dir` + empty `sessions` so a sender can address a cold agent.

## Seeding / versioning propagation

Edit ONLY the repo copies under `/home/<user>/Projects/orbit`. Both
propagation paths fire at **deploy + boot** — never hand-edit the runtime copies.

- **Skill files** (`skills/a2a/scripts/a2a_cli.py`, `a2a_lib.py`, `SKILL.md`,
  `bootstrap.sh`) reach the runtime registry (`~/.orchestrator/skills-registry/a2a`)
  via `bundled_skills.seed_bundled_skills()` on boot. Because the registry target
  already exists (a2a is installed), `_refresh_code` OVERWRITES the `_REPO_OWNED`
  files (`SKILL.md`, `TOOL.md`, `scripts`) — it `rmtree`s `scripts/` and re-copies
  from the repo on EVERY restart — while `register.json` / `config.json` (install +
  user runtime state like `dashboard_url`) are PRESERVED. The skill cache is then
  invalidated. So the `arm` / `watch-cmd` removals land because the old `scripts/`
  dir is wiped and replaced; no manual rsync.

- **General prompt** (`agent_prompts_defaults.py`) propagates via
  `agent_prompts._seed_or_migrate` on the `v8 -> v9` bump: on next boot, on-disk
  `general.md` (v8) is older than the code's v9, so it is backed up to
  `general.md.v8.bak` and rewritten with the v9 default. `identity.md` / `custom.md`
  are untouched.

- **HARD RULES:** do not edit `~/.orchestrator/**` or `~/srv/**`; do not
  `systemctl` / restart / deploy / `git push`. Until the next deploy+restart,
  running agents keep the v8 arm instructions and the old scripts — this is
  expected; changes land on the operator's normal deploy.

## Out-of-scope / deferred

Explicitly NOT part of v2 (do not build):

- **Live delivery / push.** No warm-session nudge, no SSE-driven wake, no
  auto-revive of a cold target. `send` is enqueue-only; the human drains manually.
  (The optional SSE toast on send may stay purely as a UI hint — it does not
  deliver the message.)
- **Receipts / read-receptions / delivery-confirmation.** No ack that the target
  read or drained the mail; correlation-id reply is the only linkage.
- **Group chats / fan-out to multiple agents / broadcast.** `send` targets one
  agent (optionally one `--session`). No multi-recipient addressing.
- **Monitor / arm / watch-cmd in any form.** Fully removed; not to be reintroduced
  under a new name.
- **`orchestrator_jsonl._build_summary` / `list_sessions` changes.** Not needed;
  transcript paths are resolved via `jsonl_mod.jsonl_path(sid)`.
- **`orchestrator_a2a_gc_seed.py` changes.** The maildir-GC cron and `a2a_enabled`
  gate stay as-is.
